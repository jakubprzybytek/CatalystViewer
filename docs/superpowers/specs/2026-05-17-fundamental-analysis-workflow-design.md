# Design: Fundamental Analysis Workflow (Step Functions State Machine)

**Date:** 2026-05-17  
**Status:** Draft

---

## Overview

A new AWS Step Functions state machine — `FundamentalAnalysisStateMachine` — runs on demand and performs fundamental financial analysis on corporate bond issuers listed on the Catalyst exchange. It selects issuers by staleness (those analysed least recently are prioritised), runs the AI agent loop to gather financial data for each, computes the 6-dimension risk scorecard, stores a full markdown report in DynamoDB, and sends an email summary on completion.

The state machine is on-demand only (no scheduled trigger). It is invoked manually via the AWS Console or CLI.

---

## Input Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `issuers` | `string[]` | — | Specific issuer names to analyse. When provided, `count` is ignored. |
| `count` | `number` | `2` | Number of issuers to select when `issuers` is not provided. |

Both parameters are optional. Running with no input analyses the 2 stalest corporate issuers.

---

## State Machine Flow

```
Input: { issuers?: string[], count?: number }
  │
  ▼
SelectIssuers (Task · Lambda)
  │
  ▼
AnalyzeIssuers (Map · MaxConcurrency: 1)
  ├─ RunAgent      (Task · Lambda · 10 min timeout)
  └─ StoreResult   (Task · Lambda · 30 sec timeout)
  │
  ▼
SendAnalysisReport (Task · Lambda · 30 sec timeout)

Catch-all on any state → SendErrorReport (existing Lambda)
```

### SelectIssuers

Determines which issuers will be analysed:

1. Call `BondDetailsTable.getActive('Corporate bonds')` → collect unique `issuer` names from all active corporate bonds.
2. Scan `IssuerProfiles` for all `recordType = '#LATEST_ANALYSIS'` records → build a `Map<issuerName, performedAtTs>`.
3. For each issuer from step 1: look up `performedAtTs` (use `0` if the issuer has never been analysed).
4. Sort ascending by `performedAtTs` — never-analysed issuers (timestamp 0) go first.
5. Selection:
   - If `issuers` input parameter is provided → use those exact names (no sorting applied).
   - Otherwise → take the top `count` (default 2) from the sorted list.
6. Output: `{ selectedIssuers: string[] }`.

### AnalyzeIssuers (Map)

Iterates over `selectedIssuers` with `MaxConcurrency: 1` to respect Bedrock rate limits. Each iteration runs two sequential states:

#### RunAgent

- Input: `{ issuerName: string }`
- Runs `analyzeIssuer()` from `@core/ai/issuers/IssuerAnalysis` — the same logic as `scripts/analyze-issuer/analyze-issuer.ts`:
  - Invokes `AgentLoop` with Claude Sonnet 4.6, tools: Biznesradar, Stockwatch, WebSearch (up to 10 iterations).
  - Parses the agent's JSON output into `AgentFinancials`.
  - Computes `FundamentalScorecard` via `computeScorecard()`.
  - Builds `reportMarkdown` (see Markdown Report section below).
- Output: `{ issuerName, scorecard, agentFinancials, agentLog, reportMarkdown, modelId }`
- Timeout: 10 minutes.
- On failure: the Map state catches the error and continues to `SendAnalysisReport` with a failure record for this issuer.

#### StoreResult

- Input: `RunAgent` output.
- Calls `IssuerProfilesTable.storeAnalysis()`:
  - Writes `#ANALYSIS#<iso>` timestamped record.
  - Overwrites `#LATEST_ANALYSIS` mirror record.
- Output: `{ issuerName, performedAt, success: true }`
- Timeout: 30 seconds.

### SendAnalysisReport

Sends an email summary via SES using the existing SSM recipients parameter (`/catalyst-viewer/notifications/recipients`). The email includes:
- Which issuers were analysed and their top-level scorecard signals.
- Any issuers that failed analysis and the error reason.

Reuses the SES + SSM permissions pattern from the existing `SendReport` Lambda.

### Error Handling

A catch-all on the top-level state machine routes any unhandled error to the existing `SendErrorReport_Error` Lambda, which sends an error notification email.

## Markdown Report

The `reportMarkdown` field is a human-readable account of the full analysis run. It is stored in `DbIssuerAnalysisRecord.reportMarkdown` and generated by `buildReportMarkdown()` in `@core/ai/issuers/IssuerAnalysis`.

Structure:

```markdown
# Fundamental Analysis: {issuerName}

**Date:** {performedAt}  
**Model:** {modelId}

## Financial Data

| Year | Revenue | EBIT | Net Profit | Equity | Fin. Debt |
|------|---------|------|------------|--------|-----------|
| 2024 | ...     | ...  | ...        | ...    | ...       |

**Notes:** {agentFinancials.notes}

## Scorecard

| Dimension            | Signal  | Metrics                        |
|----------------------|---------|-------------------------------|
| Debt Burden          | 🟢      | D/E: 0.8×, Net Debt/EBITDA: 2.1× |
| ...                  | ...     | ...                            |

## Agent Steps

### Iteration 1
**Tool:** stockwatch_financials  
**Query:** "Play"  
**Result:**
<raw tool output>

### Iteration 2
...

## Token Usage

- Input: 12,345
- Output: 678
- Total: 13,023
```

All agent tool calls, tool results (including raw debug output), and token usage are included. If a tool returns an error or no data, that is recorded verbatim.

## Data Model Changes

### `DbIssuerAnalysisRecord` — add `reportMarkdown`

```ts
export type DbIssuerAnalysisRecord = {
    issuerName: string;
    recordType: string;        // "#ANALYSIS#<iso>" | "#LATEST_ANALYSIS"
    performedAt: string;       // ISO 8601
    performedAtTs: number;     // Unix ms
    modelId: string;
    scorecard?: FundamentalScorecard;
    agentFinancials?: unknown;
    agentLog?: unknown[];
    reportMarkdown?: string;   // NEW — full markdown report
};
```

### `IssuerProfilesTable` — add `getAllLatestAnalyses()`

New method scans the table for all `recordType = '#LATEST_ANALYSIS'` records in a single pass and returns a `Map<string, number>` of `issuerName → performedAtTs`. Used by `SelectIssuers` to avoid N individual GetItem calls.

```ts
async getAllLatestAnalyses(): Promise<Map<string, number>>
```

## New Core Module

**`packages/core/src/ai/issuers/IssuerAnalysis.ts`**

Encapsulates the analysis logic extracted from `scripts/analyze-issuer/analyze-issuer.ts`:

```ts
export type IssuerAnalysisDeps = {
    bedrockClient: BedrockRuntimeClient;
    tavilyClient: TavilyClient;
};

export type IssuerAnalysisResult = {
    issuerName: string;
    scorecard: FundamentalScorecard;
    agentFinancials: AgentFinancials;
    agentLog: AgentEvent[];
    reportMarkdown: string;
    modelId: string;
};

export async function analyzeIssuer(
    deps: IssuerAnalysisDeps,
    issuerName: string
): Promise<IssuerAnalysisResult>

export function buildReportMarkdown(
    issuerName: string,
    agentFinancials: AgentFinancials,
    scorecard: FundamentalScorecard,
    agentEvents: AgentEvent[]
): string
```

The `AgentFinancials` type (currently defined inline in the script) is also defined and exported here.

## New Lambda Functions

| Lambda | Handler file | Timeout | Links |
|--------|-------------|---------|-------|
| `SelectIssuers` | `packages/functions/src/issuers/selectIssuers.ts` | 60 s | `bondDetailsTable`, `issuerProfilesTable` |
| `RunIssuerAnalysis` | `packages/functions/src/issuers/runIssuerAnalysis.ts` | 10 min | — (no DynamoDB access) |
| `StoreIssuerAnalysis` | `packages/functions/src/issuers/storeIssuerAnalysis.ts` | 30 s | `issuerProfilesTable` |
| `SendAnalysisReport` | `packages/functions/src/emails/sendAnalysisReport.ts` | 30 s | SES + SSM (permissions only) |

`RunIssuerAnalysis` needs Bedrock `InvokeModel` permission and `TAVILY_API_KEY` environment variable. It does not write to DynamoDB — that is `StoreIssuerAnalysis`'s responsibility.

---

## Infrastructure (`infra/updater.ts`)

A second state machine is added to `updater.ts` alongside the existing `BondsUpdaterStateMachine`:

- New IAM role: `FundamentalAnalysisSfnRole`
- New IAM policy: `FundamentalAnalysisSfnPolicy` — allows `lambda:InvokeFunction` on the 4 new Lambdas plus the existing `SendErrorReport_Error` Lambda.
- New state machine: `FundamentalAnalysisStateMachine` — name pattern `FundamentalAnalysisStateMachine-{stage}`.
- No EventBridge scheduler (on-demand only).

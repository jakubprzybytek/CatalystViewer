# Story 4 — Update Bonds Workflow Refactor: Implementation Plan

## Current State vs. Story Requirements

The workflow has two separate notification emails today:
1. `SendClassificationNotification` — fires right after classification
2. `SendNotification` — fires only when bonds changed

The story requires a **single unified report**, sent when *either* bonds changed *or* issuers were classified.

---

## Task 1 — Add `SendReportInput` type

**File:** `packages/functions/src/issuers/index.ts`

Add a new type that combines bond update results with optional classification results (classification may not happen when no issuers are collected):

```ts
export type SendReportInput = CollectIssuersResult & Partial<Pick<ClassifyIssuersResult, 'classifiedIssuers' | 'failedIssuers'>>;
```

---

## Task 2 — Create unified Pug email template

**File:** `packages/functions/src/emails/reportNotification.pug`

A single template with two conditional sections:
- **Bonds section** — rendered only when `newBonds` or `bondsDeactivated` are non-empty
- **Issuers section** — rendered only when `classifiedIssuers` or `failedIssuers` are non-empty

Styling should be based on the existing `issuerClassificationNotification.pug` template.

---

## Task 3 — Create `sendReport` Lambda handler

**File:** `packages/functions/src/emails/sendReport.ts`

Replaces both `sendNotification.ts` and `sendClassificationNotification.ts`:
- Accepts `SendReportInput`
- Builds the unified email using the new template
- Subject: `"Catalyst Update Report"`

## Task 4 — Refactor the state machine in `infra/updater.ts`

### State machine changes

| Remove | Replace with |
|--------|-------------|
| `SendClassificationNotification` → `MajorChanges` → `SendNotification` | `ShouldSendReport` → `SendReport` |

### New `ShouldSendReport` Choice condition (covers story items 5.1 + 5.2)

```json
{ "Or": [
  { "Variable": "$.Payload.newBonds[0]", "IsPresent": true },
  { "Variable": "$.Payload.bondsDeactivated[0]", "IsPresent": true },
  { "Variable": "$.Payload.unclassifiedIssuers[0]", "IsPresent": true }
]}
```

Using `unclassifiedIssuers[0]` cleanly expresses "issuers were collected for classification" — the array is always present in the payload after `CollectUnclassifiedIssuers`, so the condition is false when the array is empty and true otherwise.

### Updated flow

```
HasUnclassifiedIssuers
  ├─ (issuers present) → ClassifyIssuers ─┐
  └─ (no issuers)    ─────────────────────┴→ ShouldSendReport
                                                ├─ (bonds or issuers) → SendReport → End
                                                └─ (nothing)          → Skip
```

### Lambda registration changes

- Add `sendReportFunction` Lambda (pointing at new handler)
- Remove `notificationSenderFunction` and `classificationNotificationSenderFunction`
- Update the IAM role resource list and `$resolve([...])` ARN list accordingly

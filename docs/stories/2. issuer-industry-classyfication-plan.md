# Implementation Plan: Issuer Industry Classification

## Overview

Add AI-powered issuer classification to the `BondsUpdaterStateMachine` Step Functions workflow. A new DynamoDB table (`IssuerProfiles`) stores one record per issuer. Three new Lambda functions handle gap detection, Bedrock classification, and email notification. The new states are inserted between `UpdateBonds` and the existing `MajorChanges` choice — the original notification path is preserved unchanged.

---

## Dependency Graph

```
Phase 1 (Storage)
    ├── Phase 2 (CollectUnclassifiedIssuers Lambda)
    ├── Phase 3 (ClassifyIssuers Lambda)
    └── Phase 4 (Email) ← no dependencies, can start immediately
            ↓
        Phase 5 (Infrastructure wiring)
```

Phases 2, 3, and 4 can be worked in parallel once Phase 1 is complete.

## Phase 1 — Storage layer

> Blocks Phase 2 and 3.

### Step 1 — `DbIssuerProfile` type + `IssuerProfilesTable` class

Create `packages/core/src/storage/issuerProfiles/IssuerProfilesTable.ts`:
- DynamoDB wrapper class using `DynamoDBDocumentClient` from `@aws-sdk/lib-dynamodb` — the standard pattern now used by all table classes (`BondDetailsTable`, `BondStatisticsTable`, `ProfilesTable`). Construct it via `DynamoDBDocumentClient.from(dynamoDBClient, { marshallOptions: { removeUndefinedValues: true } })`.
- Methods:
  - `getAll(): Promise<DbIssuerProfile[]>` — full table scan returning all classified issuers.
  - `store(profile: DbIssuerProfile): Promise<void>` — puts a single record.

Create `packages/core/src/storage/issuerProfiles/index.ts` — exports:
```ts
export type DbIssuerProfile = {
    issuerName: string;       // partition key
    industry: string;
    businessSummary: string;
    classifiedAt: number;     // Unix timestamp
    modelId: string;
};
export * from './IssuerProfilesTable';
```

### Step 2 — DynamoDB table declaration

Update `infra/storage.ts` — add:
```ts
export const issuerProfilesTable = new sst.aws.Dynamo("IssuerProfiles", {
    fields: { issuerName: "string" },
    primaryIndex: { hashKey: "issuerName" }
});
```

## Phase 2 — `CollectUnclassifiedIssuers` Lambda

> Depends on Phase 1. Parallel with Phase 3 and 4.

### Step 3 — Shared issuer types

Create `packages/functions/src/issuers/index.ts`:
```ts
export type ClassifiedIssuer = {
    issuerName: string;
    industry: string;
    businessSummary: string;
    modelId: string;
};

export type FailedIssuer = {
    issuerName: string;
    errorReason: string;
};

// Payload shape after CollectUnclassifiedIssuers
export type CollectIssuersResult = UpdateBondsResult & {
    unclassifiedIssuers: string[];
};

// Payload shape after ClassifyIssuers
export type ClassifyIssuersResult = CollectIssuersResult & {
    classifiedIssuers: ClassifiedIssuer[];
    failedIssuers: FailedIssuer[];
};
```

### Step 4 — `collectUnclassifiedIssuers` handler

Create `packages/functions/src/issuers/collectUnclassifiedIssuers.ts`:
- Input: `UpdateBondsResult` (the `$.Payload` forwarded by Step Functions).
- Scan `BondDetails` table → extract unique issuer names from all active bonds.
- Scan `IssuerProfiles` table → extract already-classified issuer names.
- Compute set difference: `uniqueIssuers - classifiedIssuers`.
- Return: `{ ...input, unclassifiedIssuers: string[] }` (merged superset).

## Phase 3 — `ClassifyIssuers` Lambda

> Depends on Phase 1. Parallel with Phase 2 and 4.

### Step 5 — `classifyIssuers` handler

Create `packages/functions/src/issuers/classifyIssuers.ts`:
- Input: `CollectIssuersResult` (has `unclassifiedIssuers`).
- **Batch cap**: process at most `MAX_ISSUERS_PER_RUN = 20` issuers per execution. If the list is longer, take the first 20 and leave the rest for the next scheduled run. This prevents a single bug or accidental table wipe from triggering hundreds of Bedrock calls at once.
- Instantiate the Bedrock client with `maxAttempts: 1` to disable automatic SDK retries on throttling — throttle errors are caught as per-issuer failures instead.
- For each issuer in the (capped) `unclassifiedIssuers`:
  - Call **AWS Bedrock Converse API** (`@aws-sdk/client-bedrock-runtime`, `ConverseCommand`).
  - Model: `anthropic.claude-haiku-4-5` (if cross-region inference profile is required in the deployed region, use `us.anthropic.claude-haiku-4-5-20251001-v1:0`).
  - Prompt instructs model to return strict JSON:
    ```json
    { "industry": "<one of: Developer | Finance | Health Services | Energy | Retail | Manufacturing | Municipal | Other>", "businessSummary": "<2-3 sentences in English>" }
    ```
  - Parse JSON response; on parse failure treat as a failure.
  - Store successful result in `IssuerProfiles` table with `classifiedAt = Date.now()` and `modelId`.
  - Catch per-issuer errors (Bedrock throttling, timeout, invalid JSON) → push to `failedIssuers` with the error message; **do not abort the loop**.
- Return: `{ ...input, classifiedIssuers: ClassifiedIssuer[], failedIssuers: FailedIssuer[] }`.

## Phase 4 — Classification notification email

> No dependencies. Can start immediately.

### Step 6 — Pug email template

Create `packages/functions/src/emails/issuerClassificationNotification.pug`:
- Follow existing `bondsUpdateReportNotification.pug` style (orange/cream colour scheme, inline CSS).
- Two conditional sections:
  - **Classified** — table with columns: Issuer, Industry, Business Summary.
  - **Failed** — table with columns: Issuer, Error Reason. Section omitted entirely when `failedIssuers` is empty.

### Step 7 — `sendClassificationNotification` handler

Create `packages/functions/src/emails/sendClassificationNotification.ts`:
- Mirror `sendNotification.ts` exactly (SSM recipients fetch, `sendEmail` call).
- Input: `{ classifiedIssuers: ClassifiedIssuer[], failedIssuers: FailedIssuer[] }`.
- Build HTML via the new pug template.
- Subject: `Issuer Classification Report`.

## Phase 5 — Infrastructure wiring

> Depends on all phases above.

### Step 8 — New Lambda function declarations

Update `infra/updater.ts` — add three new `sst.aws.Function` declarations:

**`collectUnclassifiedIssuersFunction`**:
- Handler: `packages/functions/src/issuers/collectUnclassifiedIssuers.handler`
- Linked resources: `bondDetailsTable`, `issuerProfilesTable`

**`classifyIssuersFunction`**:
- Handler: `packages/functions/src/issuers/classifyIssuers.handler`
- Timeout: `5 minutes` (hard stop against runaway execution)
- Reserved concurrency: `1` (prevents parallel executions doubling Bedrock calls)
- Linked resources: `issuerProfilesTable`
- IAM permission: `bedrock:InvokeModel` on the Haiku model ARN

**`classificationNotificationSenderFunction`**:
- Handler: `packages/functions/src/emails/sendClassificationNotification.handler`
- Timeout: `10 seconds`
- `copyFiles`: the new pug template
- IAM permissions: `ses:SendEmail`, `ssm:GetParameter` for `/catalyst-viewer/notifications/recipients` (same as existing `notificationSenderFunction`)

### Step 9 — IAM role policy update

Add the three new Lambda ARNs to the existing `BondsUpdaterSfnPolicy` `lambda:InvokeFunction` resource list.

### Step 10 — Step Functions state machine update

Insert four new states between `UpdateBonds` and `MajorChanges`:

The `ClassifyIssuers` task state must set `Retry: []` (no retries) so a Lambda failure surfaces immediately rather than re-running the capped batch and incurring duplicate Bedrock calls.

```
UpdateBonds (existing, unchanged)
    ↓
CollectUnclassifiedIssuers   [Task, 60s timeout]
    ↓
HasUnclassifiedIssuers?      [Choice]
    Yes (unclassifiedIssuers[0] IsPresent) ──→ ClassifyIssuers [Task, 300s timeout, Retry: none]
                                                    ↓
                                               SendClassificationNotification [Task, 10s timeout]
                                                    ↓
    No ─────────────────────────────────────→ MajorChanges (existing, unchanged)
                                                    ↓
                                               SendNotification / Skip (existing, unchanged)
```

**Payload flow through the chain:**

| After state | `$.Payload` keys |
|---|---|
| `UpdateBonds` | `bondsUpdated`, `newBonds`, `bondsDeactivated`, `bondsFailed` |
| `CollectUnclassifiedIssuers` | + `unclassifiedIssuers` |
| `ClassifyIssuers` | + `classifiedIssuers`, `failedIssuers` |
| `SendClassificationNotification` | (pass-through, unchanged) |
| `MajorChanges` | reads `newBonds[0]`, `bondsDeactivated[0]` — still present, unchanged ✓ |

## Cost Protection Summary

| Measure | Where | What it prevents |
|---|---|---|
| Batch cap (`MAX_ISSUERS_PER_RUN = 20`) | `classifyIssuers` Lambda | Runaway cost if IssuerProfiles table is accidentally wiped |
| Bedrock client `maxAttempts: 1` | `classifyIssuers` Lambda | SDK silently retrying throttled calls |
| Step Functions `Retry: []` on `ClassifyIssuers` | `infra/updater.ts` | SFN re-running the full batch on transient Lambda errors |
| Reserved concurrency `1` | `classifyIssuersFunction` | Parallel executions multiplying Bedrock calls |
| Lambda timeout `5 minutes` | `classifyIssuersFunction` | Infinite loop in code |
| AWS Budgets alert (manual, ~$10/month on Bedrock) | AWS Console | Last-resort spend alert outside code scope |

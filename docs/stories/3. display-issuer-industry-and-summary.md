# Story: Display Issuer Industry and Summary

## Background

Story 2 introduced the `IssuerProfiles` DynamoDB table, which stores an AI-generated `industry` label and `businessSummary` for each issuer. Currently this data is never surfaced in the frontend. The `IssuerCard` component shows the issuer name and financial metrics but has no context about what the company actually does.

---

## Goal

Enrich `IssuerCard` with two pieces of profile information:

- **Industry label** — always visible, displayed as a color-coded chip/badge below the issuer name.
- **Business summary** — shown on demand: when `businessSummary` is present, a down-chevron `IconButton` appears in the top-right corner of the card (alongside the existing `Checkbox`). Clicking it expands an inline section directly below the industry badge showing the full 2–3 sentence summary; clicking the up-chevron collapses it again. This follows the same expand/collapse pattern used in `NewBondCard`.

If no profile exists for an issuer yet (classification still pending), the card renders silently without a chip or chevron — no placeholder text, no spinner per card.

---

## Data Flow

```
IssuerProfiles DynamoDB table
    ↓ (new Lambda – getIssuerProfiles)
    ↓ (new API Gateway route  GET /api/issuers/profiles)
    ↓ (new SDK function – getIssuerProfiles)
IssuersViewer
    → merge by issuerName into IssuerReport[]
        ↓
IssuerCard  (read industry + businessSummary from IssuerReport)
```

Profile data is fetched once per page load alongside `getBondReports()` — a single parallel `Promise.all` in `BondReportsBrowser`. There is no per-card lazy loading.

Extend the existing type in `packages/web/src/components/BondReportsBrowser/issuers/index.tsx`:

```ts
export type IssuerReport = {
    // existing fields unchanged

    count: number;

    maxNominalValue: number;
```

### New SDK type

Add to `packages/web/src/sdk/index.ts` (or a new `Issuers.ts` module):

```ts
export type IssuerProfile = {
    issuerName: string;
    industry: string;
    businessSummary: string;
};
```

## Frontend Changes

### 1 — New SDK function `getIssuerProfiles()`

Create `packages/web/src/sdk/Issuers.ts`:
- `GET /api/issuers/profiles` with the same auth header pattern as `getBondReports()`.
- Returns `IssuerProfile[]`.
- On error, logs and returns `[]` so a profiles fetch failure never breaks the bonds view.

### 2 — `BondReportsBrowser` — parallel fetch

In `packages/web/src/components/BondReportsBrowser/index.tsx`:
- Add `const [issuerProfiles, setIssuerProfiles] = useState<IssuerProfile[]>([])`.
- Change the existing `getBondReports()` call to `Promise.all([getBondReports(bondType), getIssuerProfiles()])`.
- Pass `issuerProfiles` down to `IssuersViewer`.

### 3 — `IssuersViewer` — merge profiles

In `packages/web/src/components/BondReportsBrowser/issuers/IssuersViewer.tsx`:
- Accept a new prop `issuerProfiles: IssuerProfile[]`.
- Inside the `useMemo`, build a `Map<string, IssuerProfile>` keyed by `issuerName`.
- When constructing each `IssuerReport`, spread in `industry` and `businessSummary` from the map if a matching profile exists.

### 4 — `IssuerCard` — display industry badge and expandable summary

In `packages/web/src/components/BondReportsBrowser/issuers/IssuerCard.tsx`:

- Add `const [expanded, setExpanded] = useState(false)`.
- If `issuerReport.industry` is defined, render a MUI `Chip` (size `small`) below the issuer name `Typography`, aligned left.
- The chip is color-coded by industry using the `sx` prop (background color + contrasting text color). See the color mapping table below.
- If `issuerReport.businessSummary` is defined, render a MUI `IconButton` (size `small`) with `ExpandMoreOutlinedIcon` / `ExpandLessOutlinedIcon` in the top-right corner of the card, placed next to the existing `Checkbox`. Clicking it toggles `expanded`.
- Below the industry chip, add a MUI `Collapse` gated on `expanded && !!issuerReport.businessSummary`. Inside, render a `Typography` (variant `body2`) with the business summary text.

#### Industry color mapping

| `Health Services` | `#2e7d32` (green) | `#ffffff` |
| `Retail` | `#6a1b9a` (purple) | `#ffffff` |

These are proposed defaults — adjust colors as needed. The mapping lives in a small helper (e.g. `industryColor(industry: string): string`) co-located with `IssuerCard`.

## Acceptance Criteria

1. Each `IssuerCard` for which a profile exists shows the industry label as a color-coded chip below the issuer name.
2. Each such card also shows a down-chevron `IconButton` in the top-right corner (next to the `Checkbox`). Clicking it expands a section below the industry chip with the business summary text; the icon changes to an up-chevron. Clicking again collapses it.
3. Cards for issuers with no profile record render identically to today — no chip, no chevron, no empty space.
4. A failure of `GET /api/issuers/profiles` does not break the bonds view; the page loads normally with profiles simply absent.
5. Industry and summary data is fetched once per page load, not per card.

---

## Out of Scope

- Filtering or grouping bonds by industry (separate story).
- Showing profiles in the bond-level view (bonds list or bond cards).
- Re-classifying or editing profiles from the UI.

Update Bonds workflow should be reorganised and implement its logic in following steps:
1. Check if UpdateBonds steps be performed: by checking `updateBonds` flag
2. Perform UpdateBonds or skip accordingly
3. Collect Issuers for classification
3.1. Defaut behaviour: all not-classified yet Issuers are being collected for classification, but cap of 20 issuers is applied
3.2. By providing `classficationCap` option, the maximum number of Issuers can be amended, but only un-classified Issuers are collected
3.3. By providing `forceClassfication`, the cap is disabled and Issuers are collected, not only the un-classified yet
4. Issuers are being classified
5. Check if report should be sent
5.1. If new bonds are has been created, or old bonds are decomissioned (current behaviour)
5.2. If any Issuers has been collected for classification
6. Report is being sent (if needed)
6.1. Bonds section is present only if there are any new or decommisioned bonds
6.2. Issuers section is present if Issuers classification happened (successfull or not)
